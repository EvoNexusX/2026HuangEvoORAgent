An Italian transportation company needs to move some empty containers from its 6 warehouses (located in Verona, Perugia, Rome, Pescara, Taranto, and Lamezia) to major national ports (Genoa, Venice, Ancona, Naples, Bari). The container inventory at the warehouses is as follows:

| Perugia | 12 |
| Rome | 20 |

The transport is carried out by a fleet of trucks. The cost to transport each container is proportional to the distance traveled by the trucks, with a rate of 30 euros per kilometer. Each truck can carry up to 2 containers. The distances are as follows:

|  | Genoa | Venice | Ancona | Naples | Bari |
|:---:|:---:|:---:|:---:|:---:|:---:|

| Perugia | $380 \mathrm{~km}$ | $340 \mathrm{~km}$ | $165 \mathrm{~km}$ | $380 \mathrm{~km}$ | $610 \mathrm{~km}$ |

| Taranto | $1010 \mathrm{~km}$ | $840 \mathrm{~km}$ | $550 \mathrm{~km}$ | $305 \mathrm{~km}$ | $95 \mathrm{~km}$ |

Write a mathematical program to find the minimum cost transportation policy and solve it using COPTPY.

Using Agent mode (three-stage: modeling + design + code generation)
【Stage 1 - Problem Analysis & Modeling】:
 **Decision variables:**  
- \( x_{ij} \): number of containers transported from warehouse \( i \) to port \( j \), for \( i = 1,\dots,6 \) and \( j = 1,\dots,5 \). Nonnegative integer.  
- \( t_{ij} \): number of trucks used on the route from warehouse \( i \) to port \( j \), for \( i = 1,\dots,6 \) and \( j = 1,\dots,5 \). Nonnegative integer.

**Objective:**  
Minimize the total cost:  
\[
\min \; 30 \sum_{i=1}^{6} \sum_{j=1}^{5} d_{ij} \cdot t_{ij}
\]  
where \( d_{ij} \) is the distance (in km) from warehouse \( i \) to port \( j \).

**Constraints:**  
1. **Supply constraints** (containers cannot exceed available supply at each warehouse):  
   \[
   \sum_{j=1}^{5} x_{ij} \leq S_i \quad \forall i = 1,\dots,6
   \]  
   where \( S_i \) is the supply at warehouse \( i \) (Verona: 10, Perugia: 12, Rome: 20, Pescara: 24, Taranto: 18, Lamezia: 40).

2. **Demand constraints** (demand at each port must be met exactly):  
   \[
   \sum_{i=1}^{6} x_{ij} = D_j \quad \forall j = 1,\dots,5
   \]  
   where \( D_j \) is the demand at port \( j \) (Genoa: 20, Venice: 15, Ancona: 25, Naples: 33, Bari: 21).

3. **Truck capacity constraints** (each truck carries at most 2 containers):  
   \[
   x_{ij} \leq 2 \cdot t_{ij} \quad \forall i,j
   \]  
   This ensures sufficient trucks are allocated for the containers shipped.

4. **Nonnegativity and integrality**:  
   \[
   x_{ij} \geq 0, \quad t_{ij} \geq 0, \quad x_{ij}, t_{ij} \in \mathbb{Z} \quad \forall i,j.
   \]

**Model type:** Mixed Integer Linear Program (MILP) / Integer Program (IP).
【Stage 2 - Gurobi Algorithm Design】:
 ## Gurobi算法设计方案

### 1. 数据准备
- **距离矩阵**：创建6×5的二维数组 `dist`，存储每个仓库到每个港口的距离
- **供应向量**：长度6的数组 `supply`，对应每个仓库的容器数量
- **需求向量**：长度5的数组 `demand`，对应每个港口的需求量

### 2. 模型构建
```python
import gurobipy as gp
from gurobipy import GRB

# 创建模型
model = gp.Model("ContainerTransportation")

# 定义索引集
warehouses = range(6)  # 0-5 对应6个仓库
ports = range(5)       # 0-4 对应5个港口
```

### 3. 决策变量定义
```python
# x[i][j]: 从仓库i到港口j的容器运输数量（整数）
x = model.addVars(warehouses, ports, 
                  vtype=GRB.INTEGER,
                  name="x")

# t[i][j]: 从仓库i到港口j使用的卡车数量（整数）
t = model.addVars(warehouses, ports,
                  vtype=GRB.INTEGER,
                  name="t")
```

### 4. 目标函数设置
```python
# 目标函数：最小化总运输成本
model.setObjective(
    gp.quicksum(30 * dist[i][j] * t[i][j] 
                for i in warehouses for j in ports),
    GRB.MINIMIZE
)
```

### 5. 约束条件添加
```python
# 1. 供应约束：每个仓库运出的容器不超过供应量
supply_constraints = {}
for i in warehouses:
    supply_constraints[i] = model.addConstr(
        gp.quicksum(x[i][j] for j in ports) <= supply[i],
        name=f"Supply_{i}"
    )

# 2. 需求约束：每个港口接收的容器恰好等于需求量
demand_constraints = {}
for j in ports:
    demand_constraints[j] = model.addConstr(
        gp.quicksum(x[i][j] for i in warehouses) == demand[j],
        name=f"Demand_{j}"
    )

# 3. 卡车容量约束：每个路线上的容器数量不超过卡车容量的2倍
capacity_constraints = {}
for i in warehouses:
    for j in ports:
        capacity_constraints[(i, j)] = model.addConstr(
            x[i][j] <= 2 * t[i][j],
            name=f"Capacity_{i}_{j}"
        )
```

### 6. 求解参数配置
```python
# 设置MIP求解参数
model.Params.MIPGap = 0.01      # 设置最优间隙为1%
model.Params.TimeLimit = 300    # 设置时间限制为300秒
model.Params.Threads = 4        # 设置线程数为4
model.Params.OutputFlag = 1     # 显示求解日志
```

### 7. 求解与结果提取
```python
# 求解模型
model.optimize()

    # 提取目标值
    total_cost = model.objVal
    
    # 提取运输方案
    transport_plan = {}
    for i in warehouses:
        for j in ports:
            if x[i][j].x > 0:
                transport_plan[(i, j)] = {
                    'containers': x[i][j].x,
                    'trucks': t[i][j].x,
                    'cost': 30 * dist[i][j] * t[i][j].x
                }
    
    # 提取卡车使用情况
    truck_usage = sum(t[i][j].x for i in warehouses for j in ports)
    
    # 验证约束满足情况
    supply_used = [sum(x[i][j].x for j in ports) for i in warehouses]
    demand_met = [sum(x[i][j].x for i in warehouses) for j in ports]
```

Now, we need to determine 4 out of 5 workers to complete one of the four tasks respectively. Due to each worker's different technical specialties, the time required for them to complete each task varies. The hours required by each worker to complete each task are shown in Table 5-2.

【Stage 1 - Problem Analysis & Modeling】:
 **Decision variables:**  
Let \( x_{ij} \) be binary variables for each worker \( i \in \{ \text{I}, \text{II}, \text{III}, \text{IV}, \text{V} \} \) and each task \( j \in \{ A, B, C, D \} \).  
- \( x_{ij} = 1 \) if worker \( i \) is assigned to task \( j \).  
- \( x_{ij} = 0 \) otherwise.

**Objective:**  
Minimize the total working hours:  
\[
\begin{aligned}
\min \quad & 9x_{\text{I},A} + 4x_{\text{I},B} + 3x_{\text{I},C} + 7x_{\text{I},D} \\
          & + 4x_{\text{II},A} + 6x_{\text{II},B} + 5x_{\text{II},C} + 6x_{\text{II},D} \\
          & + 5x_{\text{III},A} + 4x_{\text{III},B} + 7x_{\text{III},C} + 5x_{\text{III},D} \\
          & + 7x_{\text{IV},A} + 5x_{\text{IV},B} + 2x_{\text{IV},C} + 3x_{\text{IV},D} \\
          & + 10x_{\text{V},A} + 6x_{\text{V},B} + 7x_{\text{V},C} + 4x_{\text{V},D}.
\end{aligned}
\]

**Constraints:**

1. **Task coverage:** Each task must be assigned to exactly one worker.  
   \[
   \begin{aligned}
   x_{\text{I},A} + x_{\text{II},A} + x_{\text{III},A} + x_{\text{IV},A} + x_{\text{V},A} &= 1, \\
   x_{\text{I},B} + x_{\text{II},B} + x_{\text{III},B} + x_{\text{IV},B} + x_{\text{V},B} &= 1, \\
   x_{\text{I},C} + x_{\text{II},C} + x_{\text{III},C} + x_{\text{IV},C} + x_{\text{V},C} &= 1, \\
   x_{\text{I},D} + x_{\text{II},D} + x_{\text{III},D} + x_{\text{IV},D} + x_{\text{V},D} &= 1.
   \end{aligned}
   \]

2. **Worker capacity:** Each worker can be assigned to at most one task (allowing one worker to remain unassigned).  
   \[
   \begin{aligned}
   x_{\text{I},A} + x_{\text{I},B} + x_{\text{I},C} + x_{\text{I},D} &\leq 1, \\
   x_{\text{II},A} + x_{\text{II},B} + x_{\text{II},C} + x_{\text{II},D} &\leq 1, \\
   x_{\text{III},A} + x_{\text{III},B} + x_{\text{III},C} + x_{\text{III},D} &\leq 1, \\
   x_{\text{IV},A} + x_{\text{IV},B} + x_{\text{IV},C} + x_{\text{IV},D} &\leq 1, \\
   x_{\text{V},A} + x_{\text{V},B} + x_{\text{V},C} + x_{\text{V},D} &\leq 1.
   \end{aligned}
   \]

**Variable domains:**  
\( x_{ij} \in \{0, 1\} \) for all \( i, j \).

**Model type:**  
Binary Integer Linear Programming (BILP). This is a variant of the assignment problem with an unequal number of workers and tasks.

4. **目标设置**: 构建最小化总工作时间的线性目标函数
5. **约束添加**: 
   - 任务覆盖约束（每个任务=1个工人）
   - 工人容量约束（每个工人≤1个任务）
6. **模型求解**: 调用optimize()方法求解
7. **结果解析**: 提取最优分配方案和未分配工人

## 3. 技术要点

- **变量类型**: 使用`GRB.BINARY`确保x[i,j]为0-1变量
- **目标函数**: 使用`gp.quicksum()`高效构建线性表达式
- **约束条件**: 使用循环为每个任务和工人添加约束
- **结果验证**: 检查模型状态为`GRB.OPTIMAL`后提取解

 ```python
import gurobipy as gp
from gurobipy import GRB

# 初始化模型
model = gp.Model("Worker_Task_Assignment")

# 工人集合
workers = ['I', 'II', 'III', 'IV', 'V']

# 任务集合
tasks = ['A', 'B', 'C', 'D']

# 工作时间矩阵 (workers × tasks)
cost_matrix = {
    'I': {'A': 9, 'B': 4, 'C': 3, 'D': 7},
    'II': {'A': 4, 'B': 6, 'C': 5, 'D': 6},
    'III': {'A': 5, 'B': 4, 'C': 7, 'D': 5},
    'IV': {'A': 7, 'B': 5, 'C': 2, 'D': 3},
    'V': {'A': 10, 'B': 6, 'C': 7, 'D': 4}
}

# 创建二进制决策变量字典
x = model.addVars(
    workers, 
    tasks, 
    vtype=GRB.BINARY,
    name="x"
)

# 设置目标函数：最小化总工作小时数
model.setObjective(
    gp.quicksum(
        cost_matrix[i][j] * x[i, j] 
        for i in workers 
        for j in tasks
    ),
    GRB.MINIMIZE
)

# 添加约束：每个任务必须恰好分配给一个工人
for j in tasks:
    model.addConstr(
        gp.quicksum(x[i, j] for i in workers) == 1,
        name=f"Task_{j}_Coverage"
    )

# 添加约束：每个工人最多分配一个任务
for i in workers:
    model.addConstr(
        gp.quicksum(x[i, j] for j in tasks) <= 1,
        name=f"Worker_{i}_Capacity"
    )

# 求解模型
model.optimize()

# 检查求解状态并输出结果
if model.status == GRB.OPTIMAL:
    print(f"最优目标值（总工作时间）: {model.objVal}")
    print("\n分配方案:")
    assignment = {}
    for i in workers:
        for j in tasks:
            if x[i, j].X > 0.5:  # 判断变量是否为1
                assignment[j] = i
                print(f"  任务 {j} → 工人 {i}")
    
    # 识别未分配的工人
    assigned_workers = set(assignment.values())
    unassigned = [i for i in workers if i not in assigned_workers]
    if unassigned:
        print(f"\n未分配的工人: {', '.join(unassigned)}")
else:
    print("未找到最优解")
```
