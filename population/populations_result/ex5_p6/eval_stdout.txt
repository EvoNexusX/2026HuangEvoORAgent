=============== 问题 7 ==================
An Italian transportation company needs to move some empty containers from its 6 warehouses (located in Verona, Perugia, Rome, Pescara, Taranto, and Lamezia) to major national ports (Genoa, Venice, Ancona, Naples, Bari). The container inventory at the warehouses is as follows:

| Verona | 10 |
| Perugia | 12 |
| Rome | 20 |

|:---:|:---:|:---:|:---:|:---:|:---:|

| Taranto | $1010 \mathrm{~km}$ | $840 \mathrm{~km}$ | $550 \mathrm{~km}$ | $305 \mathrm{~km}$ | $95 \mathrm{~km}$ |

Write a mathematical program to find the minimum cost transportation policy and solve it using COPTPY.

使用智能体模式（建模->设计->代码生成）
【阶段1 - 问题建模】:
 我们考虑一个运输问题，需要将空集装箱从仓库运至港口，最小化总运输成本。成本与运输距离和集装箱数量成正比，每公里每集装箱30欧元。每辆卡车最多可装载2个集装箱，但成本按集装箱计算，因此卡车容量仅影响可行性，不改变成本结构。供应总量大于需求总量，因此部分集装箱可留在仓库。

**数学模型**

- **集合**：
  - 仓库集合 \( I = \{1,2,3,4,5,6\} \)，分别对应 Verona, Perugia, Rome, Pescara, Taranto, Lamezia。
  - 港口集合 \( J = \{1,2,3,4,5\} \)，分别对应 Genoa, Venice, Ancona, Naples, Bari。

- **参数**：
  - 仓库供应量 \( s_i \)（集装箱数）：
    \[
    s = [10, 12, 20, 24, 18, 40]
    \]
  - 港口需求量 \( d_j \)（集装箱数）：
    \[
    d = [20, 15, 25, 33, 21]
    \]
  - 距离 \( \text{dist}_{ij} \)（公里）：来自给定表格。
  - 单位运输成本 \( c_{ij} = 30 \cdot \text{dist}_{ij} \)（欧元/集装箱）。

- **决策变量**：
  - \( x_{ij} \geq 0 \)：从仓库 \( i \) 运到港口 \( j \) 的集装箱数量，\( \forall i \in I, j \in J \)。

- **目标函数**：
  最小化总运输成本：
  \[
  \min \sum_{i \in I} \sum_{j \in J} c_{ij} x_{ij}
  \]

- **约束条件**：
  1. **供应约束**：每个仓库运出的集装箱总数不超过其供应量。
     \[
     \sum_{j \in J} x_{ij} \leq s_i, \quad \forall i \in I
     \]
  2. **需求约束**：每个港口收到的集装箱总数等于其需求量。
     \[
     \sum_{i \in I} x_{ij} = d_j, \quad \forall j \in J
     \]
  3. **非负性**：\( x_{ij} \geq 0, \forall i \in I, j \in J \)。

由于参数均为整数且约束矩阵为全单模，线性规划松弛必得整数解，因此无需显式声明整数约束。该模型可直接转化为 Gurobi 或其他优化求解器的代码。
【阶段2 - 算法设计】:
 以下是基于给定数学模型的 **Gurobi 求解算法设计方案**，涵盖变量定义、约束添加、目标函数设置和结果提取等关键步骤。本设计采用 **Python API** 实现。

## 1. 算法设计概述

- **模型类型**：线性规划（Linear Programming, LP）
- **求解器**：Gurobi Optimizer
- **关键特性**：由于所有参数均为整数且约束矩阵为全单模（Totally Unimodular），线性规划松弛即可得到整数解，因此无需显式声明整数约束。
- **输出**：最优运输方案（各路径运输量）及最小总运输成本。

## 5. 完整算法结构

```python
import gurobipy as gp
from gurobipy import GRB

def solve_container_transport():
    # 1. 数据定义
    I = range(6)  # 仓库索引
    J = range(5)  # 港口索引
    
    s = [10, 12, 20, 24, 18, 40]
    d = [20, 15, 25, 33, 21]
    
    dist = [...]
    
    # 2. 创建模型
    model = gp.Model("ContainerTransport")
    
    # 3. 定义变量
    x = model.addVars(I, J, lb=0.0, name="x")
    
    # 4. 目标函数
    model.setObjective(
        gp.quicksum(30 * dist[i][j] * x[i, j] for i in I for j in J),
        GRB.MINIMIZE
    )
    
    # 5. 约束
    # 供应约束
    for i in I:
        model.addConstr(
            gp.quicksum(x[i, j] for j in J) <= s[i],
            name=f"Supply_{i}"
        )
    
    # 需求约束
    for j in J:
        model.addConstr(
            gp.quicksum(x[i, j] for i in I) == d[j],
            name=f"Demand_{j}"
        )
    
    # 6. 求解
    model.optimize()
    
    # 7. 结果提取
    if model.status == GRB.OPTIMAL:
        print(f"最小总成本: {model.objVal:.2f} 欧元")
        print("\n运输方案（仅显示运输量>0的路径）:")
        for i in I:
            for j in J:
                if x[i, j].x > 1e-6:
                    print(f"  仓库{i+1} -> 港口{j+1}: {x[i, j].x:.0f} 个集装箱")
        
        print("\n各仓库剩余集装箱:")
        for i in I:
            shipped = sum(x[i, j].x for j in J)
            remaining = s[i] - shipped
            print(f"  仓库{i+1}: {remaining:.0f}")
    else:
        print("未找到最优解")

if __name__ == "__main__":
    solve_container_transport()
```

1. **变量类型**：虽然问题是整数规划，但由于全单模性，使用连续变量即可获得整数解，这简化了模型。
2. **单位**：成本为欧元，距离为公里。
3. **卡车容量**：问题中提到卡车容量（最多2箱）仅影响可行性，但模型中未显式约束，因为成本结构与集装箱数量成正比，且供应充足。若需考虑整数车次，可修改模型。
4. **扩展性**：此设计易于扩展到更多仓库或港口，只需更新相应集合和参数即可。

```python
# 添加二进制变量 x[i, j]
x = model.addVars(workers, tasks, vtype=GRB.BINARY, name='x')
```

##### 步骤4：设置目标函数
- 目标：最小化总工作时间。
- 使用 `quicksum` 高效求和。

```python

    quicksum(time[i, j] * x[i, j] for i in workers for j in tasks),
    GRB.MINIMIZE
)
```

##### 步骤5：添加约束条件
- 任务约束：每个任务恰好由一名工人完成。
- 工人约束：每个工人最多承担一项任务。

```python
for j in tasks:
    model.addConstr(

        name=f'task_{j}'
    )

    model.addConstr(
        quicksum(x[i, j] for j in tasks) <= 1,
        name=f'worker_{i}'
    )
```

```python
model.optimize()
```

##### 步骤7：提取结果
- 检查求解状态：若为最优，则输出目标值和分配方案。
- 遍历变量，判断取值是否大于0.5（二进制变量在求解后可能为浮点数）。

```python
if model.status == GRB.OPTIMAL:
    print(f'最优总工作时间: {model.objVal} 小时')
    print('\n分配方案:')
    # 按工人输出
    for i in workers:
        assigned = False
        for j in tasks:
            if x[i, j].x > 0.5:
                print(f'工人 {i} -> 任务 {j} (时间: {time[i, j]} 小时)')
                assigned = True
        if not assigned:

    # 按任务输出（可选）

    for j in tasks:
        for i in workers:
            if x[i, j].x > 0.5:
                print(f'任务 {j} -> 工人 {i}')
else:
    print('未找到最优解')
```
